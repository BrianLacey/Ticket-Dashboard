Ticket Dashboard created using the MERN stack with Typescript:
MongoDB
ExpressJS
ReactJS
NodeJS

Additional libraries used:
Front End:
Material UI React for UI library and icons.
TailwindCSS for styling.
ESLint for linting/code correction.

Back End:
cors for cross-origin handling.
yup for API schema validation.
concurrently, dotenv, and tsx as dev dependencies for running the app locally and conveiently through the devleopment process.

For instructions on Front End setup to run locally, please see the README under/client, or click [here](https://github.com/BrianLacey/Ticket-Dashboard/tree/main/client#readme).
For instructions on Back End setup to run locally, please see the README under/server or click [here](https://github.com/BrianLacey/Ticket-Dashboard/blob/main/server/README.md).

Deployed using Vercel for the Front End, Render for the Back End, and Atlas for the database.
See https://ticket-dashboard-sage.vercel.app to view the deployed, functional version.

This assumes you're already have a familiarity with software development. The README under both client and server are meant to be a simple walkthrough in order to get the app running locally end-to-end; There are no troubleshooting steps available.  

It is a monolithic architecture in the interest of quickness and tidy packaging for a small app. A proper microservice architecture is of course the modern standard and can be built out much more robustly.

With more time definitely testing would be implemented for general resilience and to address potential edge cases I as a one-man developer did not encounter. I also would have put more thought and energy into the general aesthetic design of the app. It's not what I call ugly or off-putting, but the cards are not always equal size depending on factors like the size of the description, and the filter shrinks portions of the table that don't have cards.  

Overall I believe I have demonstrated my ability to build a fully functional full stack app from the ground up, up to and including deployment for shipping a product. I'm happy to demo and take questions explaining my processes.
